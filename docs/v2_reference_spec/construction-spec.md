# 建設與破壞規格（Construction & Destruction Spec）

> 來源：PTT Old-Games 板《球球寶貝（mona-mona）攻略文-基礎篇》by BugofBook
> 對應原文段落：六

---

## 1. 概覽

本規格涵蓋四種可建設物的建造與破壞：

| 建物 | 用途 | 涵蓋規格 |
|---|---|---|
| 房屋 | 領土、稅收、部隊生產 | 詳見 `house-system-spec.md` |
| 田地 | 領土、稅收倍率 | 詳見 `house-system-spec.md`（由房屋擴張產生） |
| 橋樑 | 跨越河流/岩漿 | 本規格 §2 |
| 柵欄 | 阻擋移動、切斷連通性 | 本規格 §3 |

破壞建築的共通規則見 §4。

---

## 2. 橋樑（Bridge）

### 2.1 建造條件

| 條件 | 規則 |
|---|---|
| 地形 | 只能蓋在**河流**或**岩漿**上 |
| 部隊 | 需要部隊在橋樑兩端（或其中一端，攻略未明說）發起建造 |
| 我方建造費用 | **2000 金** |
| 敵方拆橋費用 | **500 金**（拆敵方的橋，比自己造便宜得多） |
| 建造時間 | **2 ticks** |
| 部隊損失 | 建造期間每 tick 損失 **10 人** |

> ⚠️ 「敵方建設橋梁需要 500 金」這句攻略中的意思可能是：**拆掉敵方橋梁時，只需 500 金**（因為攻略提到「敵方建設」一詞較模糊，建議實作前驗證遊戲行為）。
> 另一種詮釋：在敵方領土上強行架橋只需 500 金，是入侵減稅機制。建議實作時兩種都先支援，靠 feature flag 切換。

### 2.2 戰術價值

- 河流和岩漿是天然屏障，橋樑是唯一穿越方式
- 部隊只能走自己建的橋（攻略沒明說敵方部隊能否走我方橋；建議：**所有部隊都能走任何橋**）

---

## 3. 柵欄（Fence）

### 3.1 建造條件

| 條件 | 規則 |
|---|---|
| 地形 | **所有部隊能行走的地方**都能蓋 |
| 費用 | **500 金** |
| 建造時間 | **5 ticks** |
| 部隊損失 | 建造期間每 tick 損失 **10 人** |

### 3.2 柵欄效果

- 阻擋部隊移動（敵我皆然？攻略沒明說，建議：**敵我皆阻擋**，這樣才會出現「自己被自己柵欄圍住」的策略陷阱）
- **切斷稅收連通性**：稅收路徑經過柵欄時視為不連通（見 `house-system-spec.md` §3.2）

### 3.3 戰術價值

- 廉價（500 金）、可建造在任何陸地
- 主要用於：保護核心房屋區、誘導敵軍路徑、切斷敵方稅收網

## 4. 破壞建築

### 4.1 通用規則

可破壞的建築：**房子、田地、橋樑、柵欄、城堡、怪物巢穴**

```
破壞以 tick 為單位
每 tick 攻擊部隊固定損失 10 人
破壞期間部隊無法移動或做其他事
```

### 4.2 破壞速度

| 部隊等級 | 每 tick 造成的破壞量 |
|---|---|
| S | `floor(sqrt(population / 100))` |
| M | `floor(sqrt(population / 100))` |
| L | **極高**（攻略原文：「破壞力太大沒辦法好好估算」） |

**範例計算**（S/M 級）：

| 部隊人數 | 每 tick 破壞量 |
|---|---|
| 100 | 1 |
| 400 | 2 |
| 900 | 3 |
| 2,500 | 5 |
| 10,000 | 10 |

> L 級的破壞力，攻略特別提到「攻擊橋樑和柵欄幾乎秒毀」。建議實作為 `floor(sqrt(population / 10))` 或直接 `population / 100`，M2 tune。

### 4.3 建築耐久度

攻略沒明確給出各種建築的耐久度。建議：

| 建築 | 耐久度建議 |
|---|---|
| 房子 | 等於 `population`（共用房子人數池） |
| 田地 | 1（一擊摧毀） |
| 橋樑 | 10（呼應 L 級秒毀的說法） |
| 柵欄 | 10 |
| 城堡 | 200~500（M2 tune，攻擊城堡=殺國王=即時滅國，所以要高耐久避免閃擊） |
| 怪物巢穴 | 100（呼應每 4 天 +10 隻 → 10 ticks 累積） |

## 5. 建造流程的狀態機

```
[IDLE]
  ↓ 玩家下令建造
[VALIDATING] → 失敗 → [IDLE]
  ↓ 通過驗證、扣錢
[CONSTRUCTING] (n ticks，每 tick 部隊 -10)
  ↓ 部隊人數降為 0 → [ABORTED]
  ↓ 部隊被攻擊死亡 → [ABORTED]
  ↓ 建造完成
[COMPLETED] → 建築上線
```

> 注意：建造中的部隊**仍可被攻擊**（攻略沒明說，但合理推測，因為部隊沒有特殊狀態）。M2 需決定建造中能否被取消、被攻擊時是否中止建造。

## 6. 資料模型建議

```typescript
type ConstructionType = 'BRIDGE' | 'FENCE' | 'HOUSE';

interface ConstructionTask {
  id: string;
  type: ConstructionType;
  unitId: UnitId;
  position: { x: number; y: number };
  startTick: number;
  totalTicks: number;       // BRIDGE: 2, FENCE: 5, HOUSE: 立即完成
  ticksRemaining: number;
}

interface DestructionTask {
  unitId: UnitId;
  targetType: 'HOUSE' | 'FIELD' | 'BRIDGE' | 'FENCE' | 'CASTLE' | 'MONSTER_NEST';
  targetId: string;
  startTick: number;
}

interface Building {
  id: string;
  type: ConstructionType;
  owner: PlayerId | null;   // 柵欄、橋樑可能無主
  position: { x: number; y: number };
  durability: number;
  maxDurability: number;
}
```

## 7. 單元測試清單

### 橋樑
- [ ] 在土地上嘗試建橋 → 失敗
- [ ] 在河流上建橋、有 2000 金 → 成功，扣 2000
- [ ] 建造中每 tick 部隊 -10
- [ ] 部隊人數歸零 → 建造取消
- [ ] 建造完成後第 3 tick → 橋可通行

### 柵欄
- [ ] 河流上嘗試建柵欄 → 失敗
- [ ] 土地上建柵欄、有 500 金 → 成功
- [ ] 5 tick 後完成
- [ ] 完成後切斷我方稅收連通性 → 經過柵欄的房子變 disconnected

### 破壞
- [ ] S 級 100 人攻擊房子 → 每 tick 破壞 1，部隊 -10
- [ ] M 級 2500 人攻擊柵欄 → 每 tick 破壞 5
- [ ] L 級 10000 人攻擊橋樑 → 1~2 tick 內摧毀
- [ ] 破壞城堡 → 該國國王被打倒（觸發 §1.2 of victory-conditions-spec.md）

## 8. M2 待決定議題

1. **「敵方建設橋梁 500 金」的真實語意**：拆橋優惠 vs 入侵架橋優惠
2. **柵欄是否阻擋自己部隊**：建議「是」，避免遊戲變太簡單
3. **L 級破壞力公式**：原文沒給，建議 `floor(sqrt(population / 10))` 起步
4. **各建築耐久度**：田地、橋樑、柵欄、城堡、巢穴的具體數值
5. **建造中部隊被攻擊**：是否取消建造？是否歸還已扣金錢？
