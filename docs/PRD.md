## 1. 願景與背景

重製日本免費小品《国家大作戦》(lm_exp, 2005 年前後)，以 Web 技術交付。45° 斜俯視棋盤、像素風角色、即時 tick 戰棋。**核心循環**：領地自然成長 → 拖曳派兵擴張領土 / 圍攻敵格 → 佔領敵方主城獲勝。**目標體感**：單場數分鐘到十餘分鐘內結束的中節奏對 AI 戰。

## 2. 技術取向

本作為 **2D 格狀即時戰棋**，45° iso 只是視覺投影（非 3D 空間）。所有玩法都在 2D 格網上發生。**前端技術棧、架構鐵則、模組職責與「規格 → 實作」對應一律見 [`CLAUDE.md`](../CLAUDE.md)（§2 技術棧、§3 檔案結構與實作對應）**，本文件不重述。

## 3. 名詞表

- **Tick**：遊戲時鐘最小單位，2 秒 / tick（變速時間隔等比縮短）。所有產兵、移動、戰鬥都在 tick 邊界結算。
- **格（Tile）**：棋盤上的一格，帶地形、是否主城、駐軍與「領土歸屬」。格本身不直接持有「擁有者 / 兵力」——由其駐軍與最後 claim 的勢力決定。
- **駐軍（Occupant）**：駐紮在某格的單一勢力單位群（勢力 + 兵力）。**不變式**：一個格至多一個勢力在場——單位永不與敵共格（戰鬥一律跨邊發生，§4.6）。
- **行軍縱隊（Marching column）**：派遣後沿路徑移動的暫態單位群，獨立於格上的駐軍；抵達後依 §4.5 規則進駐或發動圍攻。
- **圍攻指令（Siege order）**：一筆跨邊圍攻關係——己方前進駐點（staging 格）→ 相鄰目標格。帶縱隊**自身兵力**（不寄生 staging 駐軍）與「目標之後尚待攻佔的剩餘路線」。戰鬥逐筆結算（§4.6）。
- **Tier（階級）**：Soldier / Knight / Queen / King，由兵力即時推導（§4.4），**只用於顯示與升級語意，不參與戰鬥數值**。
- **主城（Castle）**：每個勢力的核心格，記錄原主勢力。原主在自己主城上失去駐軍即敗北（§7）。主城不產兵、不自然回血（v2.6）。
- **房屋（House）**：建在我方領地格上的建築（如同主城，是格的旗標而非獨立實體），會累積**人口**、依稅率產**金錢**、人口達門檻時生兵；為遊戲的**唯一兵源**（§4.3）。敵方攻佔該格即**夷平**（房屋與人口歸零）。
- **金錢 / 國庫（Gold）**：每勢力的資源池，用途為**建造房屋**（§4.3）。由房屋稅收累積；開局給予起始金錢。
- **稅率（Tax rate）**：每勢力 0–30%。房屋稅收 = ⌊人口 × 稅率⌋ 金錢進國庫；稅率越高金錢越多、但人口成長越慢（30% = 零成長）。玩家可調、AI 依難度固定（§5.3）。
- **人口（Population）**：房屋的內部數值，隨周圍我方領地成長，同時驅動產稅與產兵；與格上駐軍兵力**分開計算**。
- **領土歸屬 / claim**：格的「最後持有 / 走過的勢力」。駐軍進駐即設為該勢力；駐軍被殲滅後仍**保留**（供 break 判定）。空格的染色歸屬由它決定，但**不授予派遣權**（派遣來源必須有自家駐軍）。
- **格的歸屬**：1 個駐軍 → 該勢力；0 駐軍 → 領土歸屬（可能無主）；多勢力同格（戰鬥不變式下不會發生）→ 無歸屬。

## 4. 玩法核心

### 4.1 棋盤與勢力

- **可選棋盤大小**：`N×N` 方格，玩家可選 **11 / 15 / 19 / 27**（預設 **19**；於開場 Start Menu 選定，按 Start = 以該尺寸開新局，§6.2.1）。`?size=N` URL 參數降為 Start Menu 的初始預設值（deep-link 用）。棋盤 fit-to-viewport 自動縮放。
- **可選地圖外形**：**Plateau / Island / Coast**（預設 **Plateau**；於 Start Menu 選定，§6.2.1）。Plateau = 整塊抬升陸地（懸崖邊緣）；Island = 同陸塊外加裝飾海環；Coast = 程序生成、**四向對稱**的沿岸海域（把邊緣格刻成水），外形不規則但四角主城恆留陸地半島、地圖恆連通。Plateau / Island 不改地形（純渲染）；Coast 改地形（見 §4.7）。`?shape=` URL 參數降為 Start Menu 的初始預設值。
- **隨機地圖**：每次按 Start 取隨機種子，地形（水 / 山 / 林分佈）與起伏地表皆不同；`?seed=N` 可指定重現某張地圖。（headless playtest / 場景仍用固定種子，不受影響。）
- 主城恆置於四角，4 個勢力各佔一角：**Tokugawa / Takeda / Oda / Uesugi**。玩家固定操作 **Tokugawa**。
- 中立據點（Neutral 山賊）置於中央：不產兵、不行動，可被任何勢力佔領（佔領後變為己方一般領地）。
- **起始兵力與經濟**：各勢力主城 = **3**（Soldier）；中立據點 = **3**；主城旁置一座**種子房屋**（起始人口見 `economy.ts`）以啟動經濟；其餘格為空。各勢力另有**起始金錢**（足建約 2–3 座房屋）。起始 3 < Knight 閾值 5 → 開局是 Soldier 對 Soldier，戰力對稱。**駐軍不再自我複製**——兵源全來自房屋產兵（§4.3）。

### 4.2 即時 Tick 引擎與結算順序

- 全域時鐘以 2 秒 / tick 推進。HUD 顯示當前 tick 數與下一 tick 倒數。支援暫停 / 繼續 / 變速（1x / 2x / 3x / 4x）。
- **Tick 編號約定**：Tick 0 為初始狀態（僅渲染、無結算）。Tick 1 起執行結算。
- **經濟日**：經濟階段不是每 tick 跑，而是每隔 `ECONOMY_INTERVAL_TICKS` tick（一個「經濟日」）結算一次，讓成長 / 稅收 / 產兵數值可讀、節奏偏策略。
- **每個 tick 固定結算順序**：

  1. **AI 行動**：rule 模式且輪到評估的非玩家勢力跑規則狀態機派兵 / **建造房屋**（§5）；idle / scripted 勢力略過。
  2. **移動與抵達**：行軍縱隊前進一格、抵達結算（進駐己方格 / 建立圍攻指令 / 同陣營合併，§4.5）。
  3. **經濟（僅經濟日）**：依序 ①房屋人口成長 → ②收稅進國庫 → ③人口達門檻產兵（§4.3）。置於戰鬥**之前**，讓新生小隊當 tick 即可當增援。
  4. **戰鬥**：逐筆圍攻指令跨邊結算 + break→capture（§4.6）；殲滅歸零的駐軍、移除完成 / 失效的圍攻指令；**攻佔的房屋格即夷平**（§4.3 / §4.6）。
  5. **敗北結算**：主城上無原主駐軍的勢力判敗，殘兵轉中立、其行軍縱隊與圍攻指令一併消滅（§7.3）。
  6. **tick 前進**、勝負判定（§7）。

### 4.3 兵源：房屋經濟（House economy）

**駐軍不再自我複製**（v2.5 的「每 tick +1」已退場）。唯一兵源是**房屋**：花金錢建造 → 人口成長 → 產稅（金錢）＋達門檻產兵。金錢的去處是**蓋更多房屋**，故金錢間接轉化為兵力。經濟階段於**經濟日**（§4.2）結算。

- **建造（金錢 → 房屋）**：勢力在符合以下條件的格花 `HOUSE_COST` 金錢建房屋——**為我方領地**（derivedOwner = 該勢力）、**該格有我方駐軍**（建造者）、**非主城**、**尚無房屋**、**可建地形**（平原 / 林；山 / 水不可）、**周圍 8 格（Moore-8）無自家房屋**（房屋至少相隔 2 格，避免一片地被房屋鋪滿）、且**國庫 ≥ `HOUSE_COST`**。建造時把建造者半數人移入房屋作**起始人口**（另半數 ceil 留守當駐軍，故新房屋至少留 1 守軍）。
- **人口成長（每經濟日）**：每座房屋 `成長 = max(MIN_GROWTH, ⌊(GROWTH_BASE + 周圍我方領地數) × (30 − 稅率%) / 30⌋)`（周圍 = Moore-8、derivedOwner = 房主）。**越多我方鄰地成長越快、稅率越高越慢**；但成長**永不歸零**——即使 30% 滿稅，存活房屋仍以最低 `MIN_GROWTH`（每天 +1）爬向產兵門檻，避免「滿稅只囤金錢、永遠不產兵」的死局。整數運算保決定論。
- **稅收（每經濟日）**：每座房屋 `金錢 = ⌊人口 × 稅率%⌋` 進房主國庫。**無連通城堡的要求**（v1 簡化；對比 v2 reference spec）。
- **產兵（每經濟日）**：人口 **≥ `SPAWN_THRESHOLD`** → 於**相鄰我方格**生成一支 `SPAWN_SIZE` 部隊、人口 **− `SPAWN_SIZE`**（無相鄰我方格則生在房屋格本身）。人口同時餵成長 / 稅 / 兵，是同一個池。
- **夷平**：敵方 break / capture 房屋格 → 房屋與其人口**消失**（§4.6）。房屋是必須防守的資產。
- **敗北勢力**的房屋變惰性（不成長 / 不產稅 / 不產兵），但不會因敗北自動夷平，仍需被攻佔才清除。
- **相對 v2 的簡化**：無田地 / 田地擴張、無城堡連通性稅收門檻、無部隊維護成本、無獨立日曆子系統。
- **數值**（`HOUSE_COST` / `SPAWN_THRESHOLD` / `SPAWN_SIZE` / `GROWTH_BASE` / `MAX_TAX_PCT` / `ECONOMY_INTERVAL_TICKS` / 起始金錢 / 種子人口）是平衡旋鈕，集中於 `src/engine/economy.ts`（工程細節單一真相，見 [`CLAUDE.md`](../CLAUDE.md) §3）；本文件只述規則、不複製數字。

### 4.4 單位、駐軍與升級

- 每個駐軍的 tier 由其兵力即時推導：

  | 兵力             | tier    |
  | ---------------- | ------- |
  | `< 5`            | Soldier |
  | `5 ≤ 兵力 < 15`  | Knight  |
  | `15 ≤ 兵力 < 30` | Queen   |
  | `≥ 30`           | King    |

- 升級 / 降級為**隱含結果**：兵力變動後 tier 即時更新，畫面顯示對應 sprite + 數字。tier **不影響戰鬥數值**（戰鬥純看兵力，§4.6）。

### 4.5 移動、派駐、路徑、行軍

#### 4.5.1 派遣手勢

- 玩家在**己方格**（有自家駐軍）按下左鍵，拖到**目標格**放開：派遣。拖曳過程畫最短路徑高亮；無路徑顯示紅色拒絕。
- **派遣比例**：25% / 50% / 75% / 100%（記憶上次選擇）。最少派遣 1 兵。
- **主城派遣下限**：主城作為來源時，無論比例**至少留 1 兵**（兵力 = 1 則無法派遣）。避免不小心清空主城被反吃。

#### 4.5.2 路徑規則

最短路徑搜尋（4-連通）。來源必須是己方格。目標若為不可通行地形（山 / 水）→ 不可派遣。**可經過格（passable）依目標類型分流**：

- **目標為己方格（補給移動）**：整條路徑的中間格都必須是**己方領土**（有自家駐軍，或己方 claim 的空格）。中立 / 無主 / 敵方格皆為牆——補給只能在自己領土內走。
- **目標為非己方格（征服行軍）**：用**忽略所有權的最短路徑**（除不可通行地形外，所有格皆可作中間格）。縱隊沿線逐格圍攻（§4.6）。
- 目標格本身永遠不需要 passable（可瞄敵格 / 中立格 / 無主格）。

#### 4.5.3 行軍與抵達

每 tick 縱隊前進一格，依**下一格**的歸屬（讀 tick 起始快照，故同 tick 抵達互不干擾）：

- **下一格己方 + 終點** → 進駐該格（併入既有同陣營駐軍或新增），縱隊消滅。
- **下一格己方 + 非終點** → 繼續前進。
- **下一格非己方** → **圍攻**：縱隊在當前前進駐點建立圍攻指令、**不踏入目標格**。涵蓋「敵方終點」與「中途某格翻出己方控制」兩種情況。
- **同陣營多縱隊同 tick 抵達同格** → 合併為單一事件：兵力加總；續行路線取剩餘步數最少者（任一以該格為終點則視為終點抵達）。

#### 4.5.4 取消行軍

玩家可在縱隊抵達終點前取消它（UI：**右鍵點擊**該行軍 sprite；左鍵已用於選取格，故取消走右鍵）。縱隊兵力就地**落在當前格**（必為己方領土）併入駐軍。

> **已知限制（觸控）**：右鍵點擊為滑鼠專屬，觸控裝置目前無對應的取消手勢。PC / 行動裝置統一單位操控列入 future scope（§9）。

### 4.6 戰鬥（cross-edge 鄰邊 + 兩階段 claim）

每筆圍攻指令逐一結算。單位永不與敵共格——戰鬥發生在己方前進駐點與相鄰目標格之間。

**傷害公式（step-function ramp）**：每 tick 基礎傷害隨開戰時長階梯式上升，`t = 當前 tick − 開戰 tick`：

```
基礎傷害(t) = 2 ^ ⌊ log₂( max(t, 1) ) ⌋
```

| t        | 0   | 1   | 2–3 | 4–7 | 8–15 | 16–31 | …   |
| -------- | --- | --- | --- | --- | ---- | ----- | --- |
| 基礎傷害 | 1   | 1   | 2   | 4   | 8    | 16    | 2^k |

依目標格狀態進入三階段之一：

**階段一 — 目標有敵 / 野單位**：

- 守軍對縱隊還擊 `min(基礎傷害, 守軍兵力)`，再按**縱隊所在格**地形減傷（§4.7）。
- `t ≥ 1` 時縱隊對守軍輸出 `min(基礎傷害, 縱隊兵力)`，再按**目標格**地形減傷。**`t = 0`（攻堅首 tick）縱隊不輸出、只挨打**——攻方需兵力優勢才打得下守軍。
- **中立**（野怪 / 敗北殘兵）**永不還擊**，但正常受擊。
- 縱隊歸零 → 攻堅失敗、指令作廢；守軍歸零 → 目標格變空（領土歸屬保留為剛死那家）。
- **多指令同打一格**：各自獨立結算；守軍歸零後同 tick 依固定順序推進 claim。

**階段二 — 目標空但敵方 claim**：**break**——縱隊花 **1 兵**把歸屬打成無主。下一 tick 才 capture。

**階段三 — 目標空且中立 / 無主**：**capture**——縱隊花 **1 兵** claim 為己方，指令完成。

**佔領後前進（征服行軍）**：capture 成功當下，縱隊以剩餘兵力**前進到剛佔下的格**：

- 剩餘路線**非空**（中繼格）→ 在該格續成一條行軍縱隊攻下一格；該格暫留為空的己方 claim。
- 剩餘路線**為空**（終點）→ 縱隊**駐紮**該格。
- 兵力耗盡 → 指令作廢，已攻下的格維持己方 claim。

> **成本與節奏**：拿有駐軍的敵格 = 戰鬥傷亡 +（break 1 + capture 1）；拿敵方空 claim 格 = 2 兵；拿中立 / 無主格 = 1 兵。被消耗的邊界兵由 self-replicate（§4.3）補回。整體在 `O(log₂(兵力))` tick 內收斂——不會出現打不死的對峙爛尾。

### 4.7 地形（terrain）

每個格有地形：**平原 / 山 / 水 / 林**（預設平原）。

- **不可通行（山 / 水）**：不能進駐、攻佔、作為中間經過格、或作派遣目標（征服行軍最短路徑自動繞過）。
- **防禦減傷（林）**：站在林的單位受到的戰鬥傷害 **×0.75**（無條件進位，傷害 ≥1 永不歸零以免卡死）。攻方縱隊受還擊按**自身所在格**地形減傷；守軍受傷按**自身所在格**地形減傷。
- **程序生成（seeded）**：由場景亂數種子決定。流程：撒佈有機地形塊（林常見、山成嶺、水稀少）→（**Coast 外形**時）把一組四向對稱的邊緣格刻成**水**（沿用既有不可通行水域，非新地形，§4.1）→ 主城與中立據點及其四鄰強制為可通行 → **連通修復**（必要時開闢平原走廊）保證所有固定點互相連通，任何主城都不會被地形封死。Coast 的刻海在固定點處理前完成，故四角主城絕不被刻、且連通與可勝照常成立。

### 4.8 幾何 / 距離 / 邊界

- **鄰格 = 4-連通**：上下左右四格，**不含對角**。路徑、戰鬥配對、地形連通全部沿用；45° iso 投影只影響渲染、不改邏輯距離。
- **距離 = Manhattan**（AI 威脅半徑 / 目標排序用）。
- **棋盤邊界**：界內有效，界外格不存在。
- **玩家派遣無 hop 上限**：只要找得到 §4.5.2 的合法路徑就能派，無論多遠。

## 5. 非玩家勢力 AI（規則狀態機）

每個非玩家勢力的控制模式由場景設定指定：

| 模式                         | 行為                                         |
| ---------------------------- | -------------------------------------------- |
| **idle**                     | 完全靜默：不評估、不派兵（主城仍自然成長）。 |
| **scripted**                 | 只在場景指定的 tick 觸發指定派遣，其餘靜默。 |
| **rule（easy/normal/hard）** | 跑規則 AI 狀態機（§5.2–§5.3）。              |

> **預設可玩場景**：玩家 Tokugawa = idle，其餘三家 = **玩家於 Start Menu 選定的難度**（easy / normal / hard，預設 **normal**，§6.2.1）——所以開局即有真正的 AI 對手。

### 5.1 評估節奏與決定論

- **同步評估（無交錯）**：所有 rule 勢力在同一批 tick 評估（每隔「評估間隔」tick，間隔依難度，§5.3），且**都讀同一份 tick 起始盤面**再合併各自行動。刻意**不交錯**——交錯會讓「較晚行動者」看到較新盤面而佔反應優勢，在進攻型 AI 下會滾成明顯的方位勝率偏差；同步評估消除先手 / 後手之分。
- **決定論**：每次評估用 (亂數種子, 勢力, tick) 推導出的亂數。**同種子 → 同決策序列**；不同種子 → 分歧。
- **無方位偏差的同分打破**：所有「挑格」決策（進攻目標、建房格，以及房屋產兵的落點）以 per-faction 亂數 / 雜湊打破同分，**不可**用棋盤座標（tile id 字典序、Map 順序）當同分鍵——否則每個勢力都偏好同一方位的格，會滾成方位勝率偏差。此偏差在經濟稀疏時（房屋間距）特別明顯。

### 5.2 規則優先序

依序嘗試、命中即停（短路）：

**① 防守 → ② 建造房屋 → ③ 進攻 → ④ 擴張 → ⑤ 集結**

> 建造置於進攻**之上**：房屋是**唯一兵源**，故有餘錢與可用建造者時，AI 先把多餘駐軍投資成經濟，再用養出的更大軍隊進攻；建造自我把關（達「每 N 格一房」或最低房屋數即停），房屋夠了自然回到進攻 / 擴張。進攻仍置於擴張之上：一旦兵力夠破敵城就 committed 進攻，而非無止境擴張空格（避免對峙僵局）；進攻自我把關（總兵力不足就跳過），早期自然降級為建房 / 擴張成長，直到能發動決定性會師。

- **① 防守**：若自家主城「防禦半徑」（Manhattan，依難度）內有敵 / 野單位 → 從**最近的己方非主城格**以 50% 增援主城。
- **② 建造房屋**：若**國庫 ≥ 建造成本**且房屋數未達「每 `housePerTiles` 格一座」（且不少於最低房屋數）比例 → 在最佳格建房屋。最佳格 = **有駐軍**（≥ 建造門檻）、**非主城 / 非已有房屋**、**無敵鄰**（不蓋在前線）、**周圍 8 格無自家房屋**（§4.3 間距）、且**周圍我方領地最多**（成長最快）的格。同分以 per-faction 亂數打破（§5.1，避免方位偏差）。
- **③ 進攻**：挑一個可抵達的敵 / 野**邊界格**（至少有一個非敵鄰格），把**多個**來源格的餘兵（每格兵力 ≥ Knight 閾值 5、各留 1 兵）經征服行軍會師過去（在「攻擊距離」內，隨棋盤大小縮放）；要求**總兵力 ≥ 守軍 × 「兵力優勢倍率」**才發動。目標排序：可破的**敵主城**優先 → 離敵主城最近 → 守軍最弱 → **同分用 per-faction 亂數打破**（避免所有勢力集火同一方位）。
- **④ 擴張**：來源 = 有餘兵的己方格（非主城兵力 ≥ 5，送「擴張比例」；主城依 tier band 保留——`<5` 不送、`5–14` 送 ⌊×0.25⌋ 且留 5、`15–29` 送 ⌊×Queen 抽成比例⌋ 且留 15、`≥30` 送 ⌊×0.5⌋）。目標 = 與己方領土相鄰的空格中、**最靠近最近敵主城者**（朝敵推進，而非隨機四散）；無存活敵主城時隨機探索。
- **⑤ 集結**（僅啟用集結的難度）：挑最強的非主城前線格，從其相鄰的己方非主城格各送 50%（至多留 1）匯集，準備下一波。

### 5.3 難度旋鈕表

三檔難度共用同一組旋鈕：

| 旋鈕                  | easy | normal | hard | 意義                                              |
| --------------------- | ---- | ------ | ---- | ------------------------------------------------- |
| 評估間隔              | 8    | 5      | 3    | 每幾 tick 評估一次（越小越勤）                    |
| 防禦半徑              | 1    | 2      | 3    | 主城周邊威脅偵測半徑（Manhattan）                 |
| 攻擊距離（×棋盤邊長） | 0.75 | 1.25   | 2.0  | 進攻掃描半徑 = round(棋盤邊長 × 此值)，隨棋盤縮放 |
| 兵力優勢倍率          | 1.5  | 1.15   | 1.0  | 發動進攻所需的兵力優勢倍率                        |
| 啟用集結              | 否   | 是     | 是   | 是否啟用集結規則                                  |
| 擴張比例              | 0.5  | 0.5    | 0.66 | 非主城來源擴張時送出的比例                        |
| 主城 Queen 抽成       | 0.2  | 0.33   | 0.4  | Queen-band 主城（15–29）抽出的比例                |
| 稅率 %（taxPct）      | 12   | 15     | 18   | AI 固定稅率（0–30；越高金錢越多但房屋成長越慢）   |
| 每格房屋比（housePerTiles） | 4 | 3 | 2 | 約每幾格領地建一座房屋（越小蓋越多 → 經濟越強；最低恆建 3 座）|

> easy = 反應慢、視野短、保守、經濟疏，玩家可輕鬆側翼；normal = 均衡基準；hard = 評估勤、攻擊距離長、抽城更兇、房屋更密。

## 6. 視覺與 UI

### 6.1 視覺風格

- **45° 斜俯視 iso**：菱形格約 64×32px，以 `(x − y, x + y)` 投影、`(x + y)` 深度排序。
- **像素風**：最近鄰縮放、整數縮放避免次像素糊化。4 勢力色 + Neutral 灰。
- **地形繪製**：**山有真實高度**——以堆疊方塊繪製，高度 = 該格深入山體的距離（邊緣 1、往內 +1、封頂 5、每單位 12px），cluster 由邊往中心升高成曲面、頂部加金字塔峰、前面畫接縫線。平原 / 水 / 林為**程式生成的像素紋理頂面**——每地形數種 dithered 菱形貼圖（最近鄰保持脆利），加上小幅抬升的方塊邊緣（水較低、呈窪地感）、水陸交界的淺色岸線、較密的草 / 樹 / 漣漪植被；地形頂面與裝飾於載入時繪製一次（地形不變），不進每 tick 重繪。**領地標示**：頂面半透明勢力色罩染 + 勢力色描邊邊框。**45° 遮擋處理**：抬高的山格若遮到後方有單位的格 → 淡化（僅山格；平原小邊緣不觸發）。
- **起伏地表**：整個地表為**離散單位高度場**（程序生成 value noise，量化為整數單位，1 單位 = 1/4 格寬）——每格頂面坐落於起伏高度，格間以土坡 pillar 接面銜接成丘陵 / 低谷（負單位 = 低谷，不積水）。山在起伏地表之上再疊高；主城等固定格鉗在基準高度以上。**水體取平**：每個連通水域以「自身與所鄰陸地的最低高度」為單一水面，故湖 / 海貼於低處、絕不浮高於四周地表。單位 / 行軍 / 路徑皆貼著地表。高度場靜態（由格座標決定，載入時算一次），不進每 tick 重繪。
- **整體地圖外形**（§4.1）：整塊棋盤呈抬升陸塊——陸地格朝鏡頭的邊（面向界外或海）落下**土石懸崖**至暗色基座，板讀作一塊立體陸地而非平面格陣（Plateau）。Island / Coast 另在陸塊外圍畫裝飾**海環**；Coast 的懸崖沿刻出的海岸線起伏。懸崖 / 基座 / 海環只由地形決定，於載入時繪製一次（靜態，不進每 tick 重繪）。
- **動畫**：戰鬥 = 互相 bump + 受擊白色 tint flash；行軍 = 0.7× 縮小 sprite 沿路徑插值 + 兵力標籤；hover 高亮 / selection outline / 派遣虛線路徑。

### 6.2 Start Menu 與 HUD 面板

#### 6.2.1 Start Menu（開場選單）

進入頁面**先顯示** Start Menu（滿版覆蓋），由玩家選定設定後按 **Start** 才開局。四項內容：

- **玩法說明（How to play）**：目標（佔領所有敵方主城獲勝 / 主城失守則敗，§7）+ **經濟摘要**（兵源來自房屋：建房 → 人口成長 → 產稅進金錢 + 產兵；金錢可蓋更多房屋；左上角金錢 + 稅率滑桿，§4.3）+ 核心操作摘要（拖曳派兵、左鍵點擊選取、右鍵取消行軍、建房按鈕 / `B`、Space 暫停 / 變速，§6.3）。內含一組 **Move / Attack 動畫示範**：以遊戲本身的 iso 投影 + 像素單位循環演示「拖曳己方單位 → 移動到己方格 / 攻擊敵方格」的核心手勢。**無障礙**：當系統回報 `prefers-reduced-motion: reduce`（含 iOS 低耗電模式 / Android 省電模式）時，示範改以**僅透明度的淡入淡出**在來源格與目標格間交替（不做大幅位移），仍傳達手勢且避免前庭不適——**不採完全靜止**（完全靜止會讓行動裝置上的示範看似卡住，issue #26）。
- **AI 難度**：**Easy / Normal / Hard**（對應 §5.3 三檔規則 AI，預設 **Normal**）；選定後套用到三家非玩家勢力。
- **棋盤尺寸**：**11 / 15 / 19 / 27**（預設 **19**，以 `?size=` 為初始值）。
- **地圖外形**：**Plateau / Island / Coast**（預設 **Plateau**，以 `?shape=` 為初始值；各外形定義與對 AI 平衡 / 地形的影響詳見 §4.1）。

按 **Start** → 以選定的尺寸 + 難度 + 外形建立新局並開始 tick（不重新整理頁面）。Start Menu 亦可從 End Screen 的 **Main Menu** 重新進入。

#### 6.2.2 HUD 與面板

桌機佈局——底部一排：Faction Panel（左）／ HUD（中）／ 派遣比例滑桿（右）；頂部：Tile Info（中）。

- **HUD（底部中央）**：當前 tick、下一 tick 倒數、暫停 + 1x/2x/3x/4x。
- **Faction Panel（底部左）**：4 勢力控制格數、總兵力、主城存活，玩家高亮。
- **Tile Info（頂部中央）**：hover 格顯示所有權 / tier / 兵力 / 是否主城。
- **End Screen**：勝利 / 敗北 + 統計 + **Restart**（同設定重開）+ **Main Menu**（回 Start Menu）（滿版覆蓋）。
- **Responsive（窄螢幕）**：HUD 移頂部中央、Tile Info 移頂部左、Faction 移底部左，避免重疊。

### 6.3 操作方案

| 操作                  | 動作                                                           |
| --------------------- | -------------------------------------------------------------- |
| 左鍵 click            | 選取格（顯示 Tile Info）                                       |
| 左鍵 hold + drag      | 從己方格派遣到目標格                                           |
| 右鍵點擊行軍 sprite   | 取消該行軍縱隊（落地併入當前格）；觸控暫無對應手勢（§9）       |
| 右鍵 hold + drag      | 平移視窗（pan）                                                |
| 滾輪 / 中鍵           | 縮放                                                           |
| 觸控雙指              | pinch 縮放 + 平移（單指保留給選取 / 派遣，雙指時暫停單指手勢） |
| 方向鍵 / WASD         | 平移視窗                                                       |
| `Space`               | 暫停 / 繼續                                                    |
| `1` / `2` / `3` / `4` | 變速                                                           |
| `R`                   | 重置攝影機                                                     |
| `Esc`                 | 取消當前拖曳派遣                                               |

- **拖曳判定**：滑鼠移動 > 5px 才算 drag；否則放開 = click。
- **指標按壓 auto-pause**：任何指標按壓期間遊戲時間自動暫停，放開後恢復（手動暫停優先：按壓中若手動暫停，放開不會自動恢復）。讓玩家從容拖曳派遣。

## 7. 勝負條件

### 7.1 勝利

- 玩家「佔領所有其他勢力的主城」（每家原主在自己主城上失去駐軍 → 該家判敗），或在所有其他勢力皆已敗北時玩家主城仍在 → 勝利畫面。

### 7.2 敗北

- 玩家主城格上找不到玩家駐軍 → 敗北畫面（不論奪城者是誰）。

### 7.3 非玩家勢力敗北

- 原主在自己主城上失去駐軍 → 判敗。處置：
  - **留存駐軍**：原地轉**中立**（被動：正常受擊、不還手、不產兵）。同格多中立合併。
  - **行軍縱隊 + 圍攻指令**：立即全部消滅（敗北 = 立即停止行動）。
- 全部非中立勢力滅 → 平局；剩一家 → 該家勝；其餘 → 對局繼續。

## 9. 範圍外（Future Scope）

- 任意 / 非方形棋盤、更多尺寸選項
- 子城 / 多城制（已保留主城旗標介面）
- 多人連線、音樂與音效、存檔 / 讀檔、自訂勢力顏色與名稱
- 行軍縱隊主動繞路（中途被切時自動 re-route，目前為退化成圍攻 / 就地併入）
- **PC / 行動裝置統一單位操控**：取消行軍目前綁右鍵（滑鼠專屬），觸控無對應手勢；需設計一套在桌機與行動裝置都順手的單位操控（例如觸控長按 / tap 取消、與選取 / 派遣手勢不衝突）。見 GitHub issue #20。
- **LLM-tier AI（僅設計，未實作）**：在 rule tier 之外加一檔由瀏覽器直連 LLM API、以非同步「腳本指令通道」回填決策（決策帶有效期限）的 AI。涉及跨來源部署、prompt caching、成本與降級策略；完整設計探索見 git 歷史中的 `docs/PRD-v1.1-section-4-draft.md`（已隨整併移除）。

## 11. 驗證與 Headless Playtest

- **自動測試**：engine 單元測試 + 跨模組整合測試（CI 回歸網）。
- **Headless playtest**：可跑單場或多場（勝率 / 場長統計）的命令列工具。場景可指定棋盤大小、初始駐軍、各勢力 AI 模式、腳本指令、亂數種子；可用 idle + 腳本指令跑引擎回歸劇本，或用 rule 難度跑 AI 對戰。CLI 用法、場景檔格式與測試檔結構見 [`CLAUDE.md`](../CLAUDE.md)（§5 測試規範、§6 執行指令）。
- **PR 驗證**：端到端跑通 + render / UI manual smoke。
