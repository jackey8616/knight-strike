## 1. 願景與背景

重製《国家大作戦》(lm_exp) / Falcom《Lord Monarch》系玩法，以 Web 技術交付。45° 斜俯視棋盤、像素風、即時 tick 戰棋。**核心循環**：以部隊**建造房屋** → 房屋**擴張田地 + 增長人口** → 連通城堡才收得到稅 → 人口溢出**產出新部隊** → 派部隊**佔領 / 圍攻 / 破壞**敵方建築 → **打倒敵國國王（攻破城堡）或令其領土全失**取勝；同時要應付**怪物巢穴**滋生的怪物與**維持費**壓力。**目標體感**：一場數分鐘到十餘分鐘、有經濟經營 + 軍事決斷張力的對 AI / 怪物戰；並支援**多關卡 campaign**（剩餘天數跨關結算）。

## 2. 技術取向

本作為 **2D 格狀即時戰棋**，45° iso 只是視覺投影（非 3D 空間）。所有玩法都在 2D 格網上發生。**前端技術棧、架構鐵則、模組職責與「規格 → 實作」對應一律見 [`CLAUDE.md`](../CLAUDE.md)（§2、§3）**，本文件不重述。引擎層（`src/engine/**`）必須**純邏輯、可 headless**（無 Pixi / DOM / GSAP），這是跑 vitest / `pnpm playtest` / AI Spectator 重播的前提。

## 3. 名詞表

- **天（day）／回合（tick）**：`tick` 是遊戲時鐘最小結算單位，**1 天 = 2 tick**。`day = floor(tick / 2)`。產兵 / 維持費 / 戰鬥傷害 / 建造進度逐 **tick** 結算；人口增長 / 田地擴張以 **day** 為節奏。
- **速度（speed）**：slow / medium / fast = 2 / 4 / 8 ticks/sec（固定常數，**只改 tick 推進的真實間隔，不改遊戲邏輯**——決定論前提，§4.2）。
- **金錢（gold）／稅率（taxRate）**：每個勢力一個國庫（gold）與一個稅率（0–30%）。稅由**連通城堡**的房屋人口依稅率產生（§4.5）；金錢支付建造費與部隊維持費（§4.8）。
- **房屋（House）**：以部隊建造的建築，佔 1 格；持有 `population`（同時當耐久），會**擴張田地**、**增長人口**、人口溢出時**產出部隊**（§4.4）。
- **田地（Field）**：由房屋擴張產生的領土格，影響人口增長速率與佔領率；耐久 1（一擊摧毀）。
- **部隊（Unit）**：在格上的人群，持有 `population`；等級 S/M/L 由人數推導（§4.6）。一國**人數最多**的部隊帶**精銳星**。
- **怪物巢穴（MonsterNest）／怪物（Monster）**：巢穴是「無稅率、不擴張、固定產出」的特殊房屋，每 4 天 +10 隻、累積到 100 生成怪物部隊（§4.9）。怪物部隊戰鬥力帶倍率（§6 預設）。
- **建築（Building：橋樑 Bridge / 柵欄 Fence）**：橋樑跨越河流 / 岩漿；柵欄阻擋移動並**切斷稅收連通性**（§4.9）。
- **稅收連通性（connectivity）**：房屋是否存在一條「全程我方田地 / 房屋、且不經柵欄」的路徑通到自家城堡（§4.5）。不連通的房屋稅率視為 0。
- **城堡（Castle）**：每個勢力的核心格。被攻破（城堡上的守軍被殲滅 / 城堡被破壞）= 國王被打倒 = 該國滅（§7）。
- **剩餘天數（remainingDays）**：跨關保留的 campaign 計分資源；每關開始 +3000，關末依經過天數 / 佔領率 / 戰鬥效率結算（§7.3）。
- **MONSTER（偽勢力）**：承載所有怪物的特殊 FactionId；無金錢 / 無稅率、不受一般敗北條件約束、不列入計分迴圈。

## 4. 玩法核心

### 4.1 棋盤與勢力

- **可選棋盤大小**：`N×N` 方格，玩家可選 **11 / 15 / 19 / 27**（預設 **19**；於開場 Start Menu 選定）。`?size=N` 為初始預設。棋盤 fit-to-viewport 自動縮放。
- **可選地圖外形**：**Plateau / Island / Coast**（預設 **Plateau**；§8）。Coast 在引擎端刻出四向對稱沿岸海域（沿用既有不可通行水域），對稱故 AI 平衡不受影響。
- **隨機地圖**：每次按 Start 取隨機種子（`?seed=N` 可重現）；headless playtest / 場景用固定種子。
- 主城恆置於四角，4 個玩家勢力各佔一角：**Tokugawa / Takeda / Oda / Uesugi**。玩家固定操作 **Tokugawa**。
- **MONSTER 偽勢力**：承載地圖預置怪物與巢穴所生怪物；不是四角玩家勢力之一，無城堡 / 金錢 / 稅率。
- **起始開局**：各勢力擁有城堡（castle）+ 起始金錢（§6 預設）+ 一支起始部隊；地圖可預置中立怪物與怪物巢穴（由場景 / 關卡檔指定）。

### 4.2 時間：day / tick / 速度 / 結算順序

- **時間單位**：1 天 = 2 tick；`day = floor(tick / 2)`。**Tick 0 為初始狀態**（僅渲染、無結算），tick 1 起結算。
- **速度**：slow / medium / fast = 2 / 4 / 8 ticks/sec。採 **fixed timestep + accumulator**：`tickMs(speed) = 1000 / rate`，accumulator 累積真實 Δt、滿一個 tickMs 就推進一 tick。**速度只改真實間隔，不改邏輯**；暫停 = accumulator 不前進。
- **決定論**：相同 (種子, 初始 state, tick 序列) → **完全相同**的最終 state（AI Spectator 重播的前提）。所有儲存量（人口 / 金錢 / 累積值）一律為**整數**（浮點只存在於暫態計算，落地前 `floor`）。
- **每個 tick 固定結算順序**（每個子系統讀 tick 起始快照、回傳 `{state, events}`，事件依序串接，§5）：

  1. **移動與抵達**：部隊沿路徑前進一格（整支移動，§4.7）；建立接觸。
  2. **戰鬥**：接觸中的配對逐 tick 互相造成傷害、打到一方殲滅（§4.7）。
  3. **建造進度**：建造中建築 +1 tick 進度、執行部隊 −10 人 / tick；完成則建築上線（§4.9）。
  4. **破壞進度**：破壞中目標依攻擊部隊等級扣耐久、攻擊部隊 −10 人 / tick（§4.9）。
  5. **巢穴累積**：每 8 tick（= 4 天）+10 隻；累積到 100 生成怪物部隊（§4.9）。
  6. **連通性重算**（僅當本 tick 房屋 / 田地 / 柵欄 / 城堡歸屬有變動才跑——dirty flag，§4.5）。
  7. **天邊界結算**（僅當本 tick 跨過 day 邊界）：人口增長 → 田地擴張 → 房屋產兵（§4.4）。
  8. **維持費**：> 2000 人部隊收維持費；國庫不足則裁員（§4.8）。
  9. **精銳星重算**：各國最大部隊標星（§4.6）。
  10. **敗北結算**：領土全失 / 國王被打倒（含怪物災難）判敗與資源轉移（§7）。
  11. **tick 前進**、勝負與 TIME_OUT 判定（§7）。

### 4.3 經濟：金錢與稅率

- 每個勢力一個 `gold` 國庫與一個 `taxRate`（0–30%，玩家可在 UI 調整、AI 由難度決定）。
- **稅收**：每天，**連通城堡**的房屋依其人口與稅率產生金錢進國庫（不連通的房屋稅率視為 0，無稅收但人口增長最快，§4.5）。稅率對人口增長的影響見 §4.4。
- **支出**：建房 100 金、建橋 2000 金、建柵欄 500 金（§4.9）；部隊維持費（§4.8）。金錢不足以支付維持費 → 裁員（§4.8）。

### 4.4 房屋：建造 / 田地擴張 / 人口增長 / 產兵

**建造房屋**（前置條件，任一不符即回對應 reason）：

| 條件 | 規則 |
| --- | --- |
| 目標格 | 必須是「土地」且未被任何房屋 / 田地 / 城堡 / 巢穴 / 橋 / 柵欄佔據（否則 `TILE_OCCUPIED`） |
| 地形 | 不可在河流 / 岩漿等不可建造地形（否則 `TILE_NOT_BUILDABLE`） |
| 周圍八格 | **我方既有房屋**的 Moore neighborhood（8 格）內不得建房（否則 `NEIGHBOR_HAS_OWN_HOUSE`）；對手 / 中立房屋不限制 |
| 金錢 | 國庫 ≥ **100 金**（否則 `INSUFFICIENT_GOLD`），建造瞬間扣除 |
| 部隊 | 必須有我方部隊位於該目標格（否則 `NO_UNIT_ON_TILE`） |

**部隊人力分配**（建造瞬間）：

```
if 建造部隊人數 ≤ 200: 新房子人數 = floor(人數 / 2);   部隊剩餘 = 人數 − 新房子人數
else:                  新房子人數 = 100;              部隊剩餘 = 人數 − 100
```
邊界：199 → 房 99 / 部隊 100；200 → 100 / 100；201 → 100 / 101；1 → 房 0 / 部隊 1（允許，房子人數 0、暫不活動直到增長）。

**田地擴張**（房屋主動，非部隊；以 day 為節奏、§6 預設每 2 天一次）：

- 把周圍八格中「非我方田地」的可轉化格轉為我方田地，**每轉一格扣房子 10 人**。
- 房子人數不足以扣除（< 10）→ 跳過該次擴張。

**人口增長**（每天）：

- 稅率 0% 時：`每日增長 = 2 + 周圍八格我方田地數`（0 田 → +2、8 田 → +10）。
- 稅率提高 → 增長變慢；**30% 時完全停滯**（不再增長）。曲線採線性 `growthFactor = (30 − taxPct) / 30`（§6 預設，待 tune）。
- **不連通城堡**的房屋稅率視為 0（§4.5），故增長最快。

**產兵**：房子人數 ≥ 100 → 產出一支 **100 人**新部隊（生成於房子相鄰的我方領土格），房子人數 −100。田地擴張與產兵**共用同一個房子人數池**。

### 4.5 稅收連通性（Castle Connectivity）

- **連通定義**：存在一條從房屋到自家城堡的路徑，路徑上每一格都是**我方田地或我方房屋**，且**不經過任何柵欄**（含我方自己的柵欄）；不可穿過他國房屋 / 田地。
- **連通 → 正常按稅率收稅；不連通 → 稅率視為 0**（無稅收、人口增長最快）。
- **重算時機**（dirty flag，§4.2 step 6）：我方 / 敵方建造或摧毀房屋、田地變動、柵欄建造 / 被破壞、領土易主（國王被打倒）、城堡狀態改變。
- **實作**：以 BFS / DFS 從每個城堡出發、標記所有可達的我方房屋；地圖不大時每次全算（大地圖再議 union-find，列入優化）。

### 4.6 單位、等級與精銳

- **等級（顯示 / 傷害量級用）**：

  | 等級 | 人數範圍 |
  | --- | --- |
  | S | 1 – 999 |
  | M | 1,000 – 9,999 |
  | L | 10,000 – 99,999 |

- **精銳星**：一國中**人數最多**的部隊額外帶「星星」標記（精銳 / 主力）；最大部隊被消滅 → 星星轉移到次大；平手 → tie-break 取較小 id。每次部隊變動後重算（§4.2 step 9）。
- 等級為 derived（由人數計算）；**戰鬥傷害量級依等級**（§4.7），實際勝負仍以人數為準。

### 4.7 移動與行軍 + 戰鬥

**移動**：部隊**整支移動**（v2 不做派遣比例切分；唯一的「分人」發生在建房，§4.4）。沿最短路徑（4-連通）逐 tick 前進一格。**可通行（passable）**：可行走地形；**河流 / 岩漿需有橋才可過**；**柵欄阻擋移動（敵我皆擋，§6 行為決議）**。路徑搜尋沿用 BFS（忽略 / 考慮所有權依目標而定，細節見 CLAUDE.md §3）。

**戰鬥觸發與結算**：

- **接觸即戰**：兩支敵對部隊一旦接觸 → **立即開戰、打到其中一方殲滅為止、不可中斷**（無法中途撤退——進戰前的判斷是策略核心）。
- **人數至上、不可合併人數**：單支對單支，人數多者必勝（多 1 人也勝）；**同陣營部隊不會把人數加總**——兩支 5000 圍攻一支 8000，仍是各自 1v1，8000 勝。逼玩家集中兵力、不能人海。
- **傷害計算**：每 tick 結算一次，依**雙方等級 + 人數**；**等級差造成的傷害差遠大於同級內人數差**。傷害公式採 §6 預設骨架（base{S:10, M:100, L:1000} × 同級內人數縮放），待 playtest tune。
- **多軍接觸配對**：一支部隊一次只跟一支敵軍交戰；旁邊無敵的友軍自動找**最近**敵軍開打；配對取「最近、平手取較小 id」（§6 預設，確保決定論）。
- **中立 / 怪物 / 野單位**：怪物戰鬥力帶倍率（§4.9 / §6）；中立 / 敗北殘兵正常受擊。

### 4.8 維持費與裁員

- **觸發**：部隊人數 **> 2000** 時，每 tick 收維持費；人數越多越貴。費用 `floor((人數 − 2000) / N)`（§6 預設 N=100，待 tune）。
- **金錢不足**：國庫 < 維持費總額 → 部隊開始**減人（裁員）**，目標降到 < 2000（不再收費）；減速與現有人數相關（§6 預設），**不直接消滅部隊**（經濟崩潰 = 大軍縮編而非瞬殺）。
- **多部隊結算**：每 tick 把所有 > 2000 人部隊維持費加總、一次從國庫扣；不夠扣則各部隊**按比例**減人。

### 4.9 建設與破壞 + 怪物

**橋樑（Bridge）**：只能蓋在**河流 / 岩漿**上；建造費 **2000 金**、耗 **2 tick**、建造期間執行部隊每 tick −10 人；完成後**所有部隊都能走任何橋**。（攻略中「敵方橋 500 金」語意未定 → §6 行為決議：MVP 不做獨立 500 機制，破壞一律走通用規則。）

**柵欄（Fence）**：所有可行走地形皆可蓋；建造費 **500 金**、耗 **5 tick**、建造期間 −10 人 / tick；效果：**阻擋部隊移動（敵我皆擋）** + **切斷稅收連通性**（§4.5）。

**破壞建築**（可破壞：房子 / 田地 / 橋 / 柵欄 / 城堡 / 巢穴）：以 tick 為單位、攻擊部隊每 tick −10 人、破壞期間部隊無法移動或做其他事。每 tick 破壞量：S/M 級 `floor(sqrt(人數 / 100))`（100→1、2500→5、10000→10）；L 級極高（§6 預設 `floor(sqrt(人數 / 10))`，待 tune）。**破壞城堡 = 國王被打倒**（§7）。耐久度（§6 預設）：房子 = 人數、田地 1、橋 10、柵欄 10、城堡 300、巢穴 100。

**建造 / 破壞狀態機**：`IDLE → VALIDATING →（通過、扣錢）CONSTRUCTING（n tick，每 tick −10）→ COMPLETED`；部隊人數歸零 / 被攻擊致死 → `ABORTED`。**行為決議（§6）**：建造中部隊仍可被攻擊；致死則 `ABORTED`、**不退款**。

**怪物巢穴與怪物**：

- 巢穴 = 無稅率、不擴張、固定產出的特殊房屋：**每 4 天（8 tick）+10 隻**；累積到 100 → 生成 **100 隻怪物部隊**（巢穴相鄰格）、累積歸 0。整數累積：`(tick − createdTick) % 8 === 0 → +10`。
- 怪物部隊：適用 S/M/L 分級；**戰鬥力帶倍率**——`effectivePopulation = 人數 × 倍率`（§6 預設 2.0）用於傷害計算，**實際扣的是真實人數**。
- **怪物 AI（§6 行為決議）**：巢穴被動（只孵化、不主動攻擊）；怪物部隊**找最近**敵方部隊 / 領土攻擊，無偏好玩家。
- **怪物殺國王（災難，與人類殺國王不同，§7）**：該國**部隊全變怪物**（owner → MONSTER）、**房屋 / 田地還原成土地**（不被繼承）、**金錢歸 0**（§6）；巢穴不受影響。

### 4.10 地形

每格有地形：**平原 / 山 / 水（河流）/ 林 / 岩漿（LAVA，新增）**。

- **不可通行（山）**：不能進駐 / 建造 / 經過 / 作派遣目標。
- **河流 / 岩漿**：不可步行，**需架橋方可通過**（§4.9）；可作為建橋目標地形。
- **防禦減傷（林，沿用）**：站在林的單位受傷 ×0.75（進位、≥1）。v2 戰鬥是否保留林減傷列為可選（§6，預設保留）。
- **程序生成（seeded）**：沿用 v1 的地形生成（撒佈有機地形塊 → Coast 刻海 → 城堡 / 巢穴及四鄰強制可通行 → 連通修復保證固定點互連），不再贅述（見 CLAUDE.md §3 / v1 邏輯）。

### 4.11 幾何 / 距離 / 邊界

- **鄰格 = 4-連通**（路徑 / 戰鬥配對 / 連通）；**Moore 8 格**用於建房排除與田地擴張。45° iso 只影響渲染。
- **距離 = Manhattan**（AI 目標排序 / 配對 tie-break 用）。
- **棋盤邊界**：界內有效，界外不存在。

## 5. 非玩家勢力 AI 與觀察事件（AI Spectator）

### 5.1 對手 AI（待完整設計）

v2_spec **未細定**對手 AI 行為；v2 的 AI 需在經濟模型上重做（建房 / 調稅 / 擴張 / 進攻 / 破壞的決策），與 v1 的 count-tier 規則機完全不同。本節先佔位：AI 沿用「難度旋鈕 + 同步評估 + 決定論（同種子同決策）」的框架（細則見 MILESTONES M12，實作前另開設計回合）。怪物 AI 行為見 §4.9 / §6。

### 5.2 AI Spectator 事件 log（跨系統）

各子系統以 **append-only event log** 記錄關鍵事件，供 spectator 重播與 AI 校準。事件不存進 `GameState`（保持 state 可雜湊、利於決定論測試與 100 場 balance run）；由呼叫端（tick → runner → spectator）累積。事件類型（至少）：

- **房屋**：`house.built` / `house.destroyed` / `house.expanded_field` / `house.spawned_unit` / `connectivity.recomputed`（含 before/after diff）。
- **戰鬥**：`combat.engaged` / `combat.damage_dealt` / `combat.unit_destroyed` / `unit.starvation` / `unit.elite_changed`。
- **建設**：`construction.started` / `construction.completed` / `construction.aborted` / `building.destroyed`。
- **怪物**：`nest.accumulated` / `monster.spawned` / `nation.consumed_by_monster`。
- **全局**：`tick.elapsed` / `day.elapsed` / `nation.defeated` / `level.completed`。

**重播保證**：同 (種子, scenario) → 同事件序列（由 §4.2 決定論保證）。

## 7. 勝負條件與關卡結算

### 7.1 國家失敗條件

1. **領土全失**：一國**既無房子也無部隊** → 該國消滅；金錢消失（不歸攻擊方）；田地保留為無主土地。
2. **國王被打倒**：城堡上守軍被殲滅 / 城堡被破壞 → 該國消滅。
   - **人類打倒**：該國金錢 / 房子 / 田地 / 部隊**全部由攻擊方繼承**。
   - **怪物打倒（災難）**：部隊變怪物、房子 / 田地還原成土地、金錢歸 0（§4.9）。
3. **時間用盡（玩家專屬）**：玩家經過天數 > 剩餘天數 → 玩家失敗。

**失敗優先序**（同 tick 多條件觸發時）：① 時間用盡（global 先查）→ ② 國王被打倒（立即觸發繼承 / 災難）→ ③ 領土全失（tick 末檢查）。

### 7.2 勝利

- 玩家打倒所有其他玩家勢力國王（或其餘皆敗）且玩家城堡仍在 → 勝利。
- 玩家城堡被攻破 / 領土全失 / 時間用盡 → 敗北。

### 7.3 關卡結算（多關卡 campaign）

`remainingDays` 跨關保留。每關**結算順序**：

```
1. remainingDays += 3000                              // 開局獎勵
2. remainingDays -= elapsed_days                      // 扣本關經過天數
3. occupation_rate = (我方房子數 + 我方田地數) / 可建房土地總數
   remainingDays -= remainingDays × (1 − occupation_rate)   // 佔領率懲罰（100% 不扣）
4. battle_efficiency = min((敵 + 怪物部隊損失總數 / 我方部隊損失人數) × 100, 600)
   remainingDays += battle_efficiency − 100           // 戰鬥效率獎勵（1:1→0、6:1→+500、我損多→負）
5. 最終 remainingDays 帶入下一關
```
