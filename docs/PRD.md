## 1. 願景與背景

重製日本免費小品《国家大作戦》(lm_exp, 2005 年前後)，以 Web 技術交付。45° 斜俯視棋盤、像素風角色、即時 tick 戰棋。**核心循環**：領地自然成長 → 拖曳派兵擴張領土 / 圍攻敵格 → 佔領敵方主城獲勝。**目標體感**：單場數分鐘到十餘分鐘內結束的中節奏對 AI 戰。

## 2. 技術取向

本作為 **2D 格狀即時戰棋**，45° iso 只是視覺投影（非 3D 空間）。所有玩法都在 2D 格網上發生。**前端技術棧、架構鐵則、模組職責與「規格 → 實作」對應一律見 [`CLAUDE.md`](../CLAUDE.md)（§2 技術棧、§3 檔案結構與實作對應）**，本文件不重述。

## 3. 名詞表

- **Tick**：遊戲時鐘最小單位，2 秒 / tick（變速時間隔等比縮短）。所有產兵、移動、戰鬥都在 tick 邊界結算。
- **格（Tile）**：棋盤上的一格，帶地形、是否主城、駐軍與「領土歸屬」。格本身不直接持有「擁有者 / 兵力」——由其駐軍與最後 claim 的勢力決定。
- **駐軍（Occupant）**：駐紮在某格的單一勢力單位群（勢力 + 兵力）。**不變式**：一個格至多一個勢力在場——單位永不與敵共格（戰鬥一律跨邊發生，§4.6）。
- **行軍縱隊（Marching column）**：派遣後沿路徑移動的暫態單位群，獨立於格上的駐軍；抵達後依 §4.5 規則進駐或發動圍攻。
- **圍攻指令（Siege order）**：一筆跨邊圍攻關係——己方前進駐點（staging 格）→ 相鄰目標格。帶縱隊**自身兵力**（不寄生 staging 駐軍）與「目標之後尚待攻佔的剩餘路線」。戰鬥逐筆結算（§4.6）。
- **Tier（階級）**：Soldier / Knight / Queen / King，由兵力即時推導（§4.4），**只用於顯示與升級語意，不參與戰鬥數值**。
- **主城（Castle）**：每個勢力的核心格，記錄原主勢力。原主在自己主城上失去駐軍即敗北（§7）。
- **領土歸屬 / claim**：格的「最後持有 / 走過的勢力」。駐軍進駐即設為該勢力；駐軍被殲滅後仍**保留**（供 break 判定）。空格的染色歸屬由它決定，但**不授予派遣權**（派遣來源必須有自家駐軍）。
- **格的歸屬**：1 個駐軍 → 該勢力；0 駐軍 → 領土歸屬（可能無主）；多勢力同格（戰鬥不變式下不會發生）→ 無歸屬。

## 4. 玩法核心

### 4.1 棋盤與勢力

- **可選棋盤大小**：`N×N` 方格，玩家可選 **11 / 15 / 19 / 27**（預設 **19**；經 `?size=N` URL 參數記憶，切換 = 開新局）。棋盤 fit-to-viewport 自動縮放。
- 主城恆置於四角，4 個勢力各佔一角：**Tokugawa / Takeda / Oda / Uesugi**。玩家固定操作 **Tokugawa**。
- 中立據點（Neutral 山賊）置於中央：不產兵、不行動，可被任何勢力佔領（佔領後變為己方一般領地）。
- **起始兵力**：各勢力主城 = **3**（Soldier）；中立據點 = **3**；其餘格為空。起始 3 < Knight 閾值 5 → 開局是 Soldier 對 Soldier，戰力對稱；主城 +1/tick 自然成長到 Knight(5) 需 2 ticks。

### 4.2 即時 Tick 引擎與結算順序

- 全域時鐘以 2 秒 / tick 推進。HUD 顯示當前 tick 數與下一 tick 倒數。支援暫停 / 繼續 / 變速（1x / 2x / 3x / 4x）。
- **Tick 編號約定**：Tick 0 為初始狀態（僅渲染、無結算）。Tick 1 起執行結算。產兵自 tick 1 起每 tick 觸發。
- **每個 tick 固定結算順序**：

  1. **AI 行動**：rule 模式且輪到評估的非玩家勢力跑規則狀態機派兵（§5）；idle / scripted 勢力略過。
  2. **移動與抵達**：行軍縱隊前進一格、抵達結算（進駐己方格 / 建立圍攻指令 / 同陣營合併，§4.5）。
  3. **產兵**：每個非戰鬥格的合格駐軍 +1（§4.3）。置於戰鬥**之前**，讓新兵當 tick 即可當增援。
  4. **戰鬥**：逐筆圍攻指令跨邊結算 + break→capture（§4.6）；殲滅歸零的駐軍、移除完成 / 失效的圍攻指令。
  5. **敗北結算**：主城上無原主駐軍的勢力判敗，殘兵轉中立、其行軍縱隊與圍攻指令一併消滅（§7.3）。
  6. **tick 前進**、勝負判定（§7）。

### 4.3 駐紮分裂與兵源（self-replicate）

每 tick，棋盤上每個格的每個駐軍若——**未被圍攻**（不是任何圍攻指令的目標格）、勢力**非中立且未敗北**、兵力 **1–99**——則 **+1**：

- **每格每 tick +1**（主城與一般領地一視同仁；殘血 1 兵也會慢慢回血）。**主城不再是特殊兵源**——只剩「失守即敗」的特殊性。
- **被圍攻的目標格凍結成長**：避免守軍在攻城下無限回血。圍攻縱隊的兵力獨立於格駐軍，其前進駐點照常成長。
- **兵力上限 100**，套在每個駐軍各自的兵力。

### 4.4 單位、駐軍與升級

- 每個駐軍的 tier 由其兵力即時推導：

  | 兵力             | tier    |
  | ---------------- | ------- |
  | `< 5`            | Soldier |
  | `5 ≤ 兵力 < 15`  | Knight  |
  | `15 ≤ 兵力 < 30` | Queen   |
  | `≥ 30`           | King    |

- 升級 / 降級為**隱含結果**：兵力變動後 tier 即時更新，畫面顯示對應 sprite + 數字。tier **不影響戰鬥數值**（戰鬥純看兵力，§4.6）。

### 4.5 移動、派駐、路徑、行軍

#### 4.5.1 派遣手勢

- 玩家在**己方格**（有自家駐軍）按下左鍵，拖到**目標格**放開：派遣。拖曳過程畫最短路徑高亮；無路徑顯示紅色拒絕。
- **派遣比例**：25% / 50% / 75% / 100%（記憶上次選擇）。最少派遣 1 兵。
- **主城派遣下限**：主城作為來源時，無論比例**至少留 1 兵**（兵力 = 1 則無法派遣）。避免不小心清空主城被反吃。

#### 4.5.2 路徑規則

最短路徑搜尋（4-連通）。來源必須是己方格。目標若為不可通行地形（山 / 水）→ 不可派遣。**可經過格（passable）依目標類型分流**：

- **目標為己方格（補給移動）**：整條路徑的中間格都必須是**己方領土**（有自家駐軍，或己方 claim 的空格）。中立 / 無主 / 敵方格皆為牆——補給只能在自己領土內走。
- **目標為非己方格（征服行軍）**：用**忽略所有權的最短路徑**（除不可通行地形外，所有格皆可作中間格）。縱隊沿線逐格圍攻（§4.6）。
- 目標格本身永遠不需要 passable（可瞄敵格 / 中立格 / 無主格）。

#### 4.5.3 行軍與抵達

每 tick 縱隊前進一格，依**下一格**的歸屬（讀 tick 起始快照，故同 tick 抵達互不干擾）：

- **下一格己方 + 終點** → 進駐該格（併入既有同陣營駐軍或新增），縱隊消滅。
- **下一格己方 + 非終點** → 繼續前進。
- **下一格非己方** → **圍攻**：縱隊在當前前進駐點建立圍攻指令、**不踏入目標格**。涵蓋「敵方終點」與「中途某格翻出己方控制」兩種情況。
- **同陣營多縱隊同 tick 抵達同格** → 合併為單一事件：兵力加總；續行路線取剩餘步數最少者（任一以該格為終點則視為終點抵達）。

#### 4.5.4 取消行軍

玩家可在縱隊抵達終點前取消它（UI：點擊該行軍 sprite）。縱隊兵力就地**落在當前格**（必為己方領土）併入駐軍。

### 4.6 戰鬥（cross-edge 鄰邊 + 兩階段 claim）

每筆圍攻指令逐一結算。單位永不與敵共格——戰鬥發生在己方前進駐點與相鄰目標格之間。

**傷害公式（step-function ramp）**：每 tick 基礎傷害隨開戰時長階梯式上升，`t = 當前 tick − 開戰 tick`：

```
基礎傷害(t) = 2 ^ ⌊ log₂( max(t, 1) ) ⌋
```

| t        | 0   | 1   | 2–3 | 4–7 | 8–15 | 16–31 | …   |
| -------- | --- | --- | --- | --- | ---- | ----- | --- |
| 基礎傷害 | 1   | 1   | 2   | 4   | 8    | 16    | 2^k |

依目標格狀態進入三階段之一：

**階段一 — 目標有敵 / 野單位**：

- 守軍對縱隊還擊 `min(基礎傷害, 守軍兵力)`，再按**縱隊所在格**地形減傷（§4.7）。
- `t ≥ 1` 時縱隊對守軍輸出 `min(基礎傷害, 縱隊兵力)`，再按**目標格**地形減傷。**`t = 0`（攻堅首 tick）縱隊不輸出、只挨打**——攻方需兵力優勢才打得下守軍。
- **中立**（野怪 / 敗北殘兵）**永不還擊**，但正常受擊。
- 縱隊歸零 → 攻堅失敗、指令作廢；守軍歸零 → 目標格變空（領土歸屬保留為剛死那家）。
- **多指令同打一格**：各自獨立結算；守軍歸零後同 tick 依固定順序推進 claim。

**階段二 — 目標空但敵方 claim**：**break**——縱隊花 **1 兵**把歸屬打成無主。下一 tick 才 capture。

**階段三 — 目標空且中立 / 無主**：**capture**——縱隊花 **1 兵** claim 為己方，指令完成。

**佔領後前進（征服行軍）**：capture 成功當下，縱隊以剩餘兵力**前進到剛佔下的格**：

- 剩餘路線**非空**（中繼格）→ 在該格續成一條行軍縱隊攻下一格；該格暫留為空的己方 claim。
- 剩餘路線**為空**（終點）→ 縱隊**駐紮**該格。
- 兵力耗盡 → 指令作廢，已攻下的格維持己方 claim。

> **成本與節奏**：拿有駐軍的敵格 = 戰鬥傷亡 +（break 1 + capture 1）；拿敵方空 claim 格 = 2 兵；拿中立 / 無主格 = 1 兵。被消耗的邊界兵由 self-replicate（§4.3）補回。整體在 `O(log₂(兵力))` tick 內收斂——不會出現打不死的對峙爛尾。

### 4.7 地形（terrain）

每個格有地形：**平原 / 山 / 水 / 林**（預設平原）。

- **不可通行（山 / 水）**：不能進駐、攻佔、作為中間經過格、或作派遣目標（征服行軍最短路徑自動繞過）。
- **防禦減傷（林）**：站在林的單位受到的戰鬥傷害 **×0.75**（無條件進位，傷害 ≥1 永不歸零以免卡死）。攻方縱隊受還擊按**自身所在格**地形減傷；守軍受傷按**自身所在格**地形減傷。
- **程序生成（seeded）**：由場景亂數種子決定。流程：撒佈有機地形塊（林常見、山成嶺、水稀少）→ 主城與中立據點及其四鄰強制為可通行 → **連通修復**（必要時開闢平原走廊）保證所有固定點互相連通，任何主城都不會被地形封死。

### 4.8 幾何 / 距離 / 邊界

- **鄰格 = 4-連通**：上下左右四格，**不含對角**。路徑、戰鬥配對、地形連通全部沿用；45° iso 投影只影響渲染、不改邏輯距離。
- **距離 = Manhattan**（AI 威脅半徑 / 目標排序用）。
- **棋盤邊界**：界內有效，界外格不存在。
- **玩家派遣無 hop 上限**：只要找得到 §4.5.2 的合法路徑就能派，無論多遠。

## 5. 非玩家勢力 AI（規則狀態機）

每個非玩家勢力的控制模式由場景設定指定：

| 模式                         | 行為                                         |
| ---------------------------- | -------------------------------------------- |
| **idle**                     | 完全靜默：不評估、不派兵（主城仍自然成長）。 |
| **scripted**                 | 只在場景指定的 tick 觸發指定派遣，其餘靜默。 |
| **rule（easy/normal/hard）** | 跑規則 AI 狀態機（§5.2–§5.3）。              |

> **預設可玩場景**：玩家 Tokugawa = idle，其餘三家 = **normal**——所以開局即有真正的 AI 對手。

### 5.1 評估節奏與決定論

- **同步評估（無交錯）**：所有 rule 勢力在同一批 tick 評估（每隔「評估間隔」tick，間隔依難度，§5.3），且**都讀同一份 tick 起始盤面**再合併各自行動。刻意**不交錯**——交錯會讓「較晚行動者」看到較新盤面而佔反應優勢，在進攻型 AI 下會滾成明顯的方位勝率偏差；同步評估消除先手 / 後手之分。
- **決定論**：每次評估用 (亂數種子, 勢力, tick) 推導出的亂數。**同種子 → 同決策序列**；不同種子 → 分歧。

### 5.2 規則優先序

依序嘗試、命中即停（短路）：

**① 防守 → ② 進攻 → ③ 擴張 → ④ 集結**

> 進攻置於擴張之上：一旦兵力夠破敵城就 committed 進攻，而非無止境擴張空格（避免對峙僵局）。進攻會自我把關（總兵力不足就跳過），故早期自然降級為擴張成長，直到能發動決定性會師。

- **① 防守**：若自家主城「防禦半徑」（Manhattan，依難度）內有敵 / 野單位 → 從**最近的己方非主城格**以 50% 增援主城。
- **② 進攻**：挑一個可抵達的敵 / 野**邊界格**（至少有一個非敵鄰格），把**多個**來源格的餘兵（每格兵力 ≥ Knight 閾值 5、各留 1 兵）經征服行軍會師過去（在「攻擊距離」內，隨棋盤大小縮放）；要求**總兵力 ≥ 守軍 × 「兵力優勢倍率」**才發動。目標排序：可破的**敵主城**優先 → 離敵主城最近 → 守軍最弱 → **同分用 per-faction 亂數打破**（避免所有勢力集火同一方位）。
- **③ 擴張**：來源 = 有餘兵的己方格（非主城兵力 ≥ 5，送「擴張比例」；主城依 tier band 保留——`<5` 不送、`5–14` 送 ⌊×0.25⌋ 且留 5、`15–29` 送 ⌊×Queen 抽成比例⌋ 且留 15、`≥30` 送 ⌊×0.5⌋）。目標 = 與己方領土相鄰的空格中、**最靠近最近敵主城者**（朝敵推進，而非隨機四散）；無存活敵主城時隨機探索。
- **④ 集結**（僅啟用集結的難度）：挑最強的非主城前線格，從其相鄰的己方非主城格各送 50%（至多留 1）匯集，準備下一波。

### 5.3 難度旋鈕表

三檔難度共用同一組旋鈕：

| 旋鈕                  | easy | normal | hard | 意義                                              |
| --------------------- | ---- | ------ | ---- | ------------------------------------------------- |
| 評估間隔              | 8    | 5      | 3    | 每幾 tick 評估一次（越小越勤）                    |
| 防禦半徑              | 1    | 2      | 3    | 主城周邊威脅偵測半徑（Manhattan）                 |
| 攻擊距離（×棋盤邊長） | 0.75 | 1.25   | 2.0  | 進攻掃描半徑 = round(棋盤邊長 × 此值)，隨棋盤縮放 |
| 兵力優勢倍率          | 1.5  | 1.15   | 1.0  | 發動進攻所需的兵力優勢倍率                        |
| 啟用集結              | 否   | 是     | 是   | 是否啟用集結規則                                  |
| 擴張比例              | 0.5  | 0.5    | 0.66 | 非主城來源擴張時送出的比例                        |
| 主城 Queen 抽成       | 0.2  | 0.33   | 0.4  | Queen-band 主城（15–29）抽出的比例                |

> easy = 反應慢、視野短、保守，玩家可輕鬆側翼；normal = 均衡基準；hard = 評估勤、攻擊距離長、抽城更兇。

## 6. 視覺與 UI

### 6.1 視覺風格

- **45° 斜俯視 iso**：菱形格約 64×32px，以 `(x − y, x + y)` 投影、`(x + y)` 深度排序。
- **像素風**：最近鄰縮放、整數縮放避免次像素糊化。4 勢力色 + Neutral 灰。
- **地形繪製**：只有**山有高度**——以堆疊方塊繪製，高度 = 該格深入山體的距離（邊緣 1、往內 +1、封頂 5、每單位 12px），cluster 由邊往中心升高成曲面、頂部加金字塔峰、前面畫接縫線。平原 / 水 / 林為平面僅以顏色區分。**45° 遮擋處理**：抬高的山格若遮到後方有單位的格 → 淡化。
- **動畫**：戰鬥 = 互相 bump + 受擊白色 tint flash；行軍 = 0.7× 縮小 sprite 沿路徑插值 + 兵力標籤；hover 高亮 / selection outline / 派遣虛線路徑。

### 6.2 HUD 與面板

桌機佈局——底部一排：Faction Panel（左）／ HUD（中）／ 派遣比例滑桿（右）；頂部：Tile Info（中）／ Map Size 選單（右）。

- **HUD（底部中央）**：當前 tick、下一 tick 倒數、暫停 + 1x/2x/3x/4x。
- **Faction Panel（底部左）**：4 勢力控制格數、總兵力、主城存活，玩家高亮。
- **Tile Info（頂部中央）**：hover 格顯示所有權 / tier / 兵力 / 是否主城。
- **Map Size 選單（頂部右）**：切換 11 / 15 / 19 / 27（= 開新局）。
- **End Screen**：勝利 / 敗北 + 統計 + 重新開始（滿版覆蓋）。
- **Responsive（窄螢幕）**：HUD 移頂部中央、Tile Info 移頂部左、Faction 移底部左，避免重疊。

### 6.3 操作方案

| 操作                  | 動作                                                           |
| --------------------- | -------------------------------------------------------------- |
| 左鍵 click            | 選取格（顯示 Tile Info）                                       |
| 左鍵 hold + drag      | 從己方格派遣到目標格                                           |
| 點擊行軍 sprite       | 取消該行軍縱隊（落地併入當前格）                               |
| 右鍵 hold + drag      | 平移視窗（pan）                                                |
| 滾輪 / 中鍵           | 縮放                                                           |
| 觸控雙指              | pinch 縮放 + 平移（單指保留給選取 / 派遣，雙指時暫停單指手勢） |
| 方向鍵 / WASD         | 平移視窗                                                       |
| `Space`               | 暫停 / 繼續                                                    |
| `1` / `2` / `3` / `4` | 變速                                                           |
| `R`                   | 重置攝影機                                                     |
| `Esc`                 | 取消當前拖曳派遣                                               |

- **拖曳判定**：滑鼠移動 > 5px 才算 drag；否則放開 = click。
- **指標按壓 auto-pause**：任何指標按壓期間遊戲時間自動暫停，放開後恢復（手動暫停優先：按壓中若手動暫停，放開不會自動恢復）。讓玩家從容拖曳派遣。

## 7. 勝負條件

### 7.1 勝利

- 玩家「佔領所有其他勢力的主城」（每家原主在自己主城上失去駐軍 → 該家判敗），或在所有其他勢力皆已敗北時玩家主城仍在 → 勝利畫面。

### 7.2 敗北

- 玩家主城格上找不到玩家駐軍 → 敗北畫面（不論奪城者是誰）。

### 7.3 非玩家勢力敗北

- 原主在自己主城上失去駐軍 → 判敗。處置：
  - **留存駐軍**：原地轉**中立**（被動：正常受擊、不還手、不產兵）。同格多中立合併。
  - **行軍縱隊 + 圍攻指令**：立即全部消滅（敗北 = 立即停止行動）。
- 全部非中立勢力滅 → 平局；剩一家 → 該家勝；其餘 → 對局繼續。

## 11. 驗證與 Headless Playtest

- **自動測試**：engine 單元測試 + 跨模組整合測試（CI 回歸網）。
- **Headless playtest**：可跑單場或多場（勝率 / 場長統計）的命令列工具。場景可指定棋盤大小、初始駐軍、各勢力 AI 模式、腳本指令、亂數種子；可用 idle + 腳本指令跑引擎回歸劇本，或用 rule 難度跑 AI 對戰。CLI 用法、場景檔格式與測試檔結構見 [`CLAUDE.md`](../CLAUDE.md)（§5 測試規範、§6 執行指令）。
- **PR 驗證**：端到端跑通 + render / UI manual smoke。
