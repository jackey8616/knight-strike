# 房屋系統規格（House System Spec）

> 來源：PTT Old-Games 板《球球寶貝（mona-mona）攻略文-基礎篇》by BugofBook
> 對應遊戲：球球寶貝 / 國家大作戰 / Lord Monarch（Falcom）
> 適用範圍：M1 引擎已實作 + M2 將補完的部分

---

## 1. 建造房屋（Build House）

### 1.1 前置條件

| 條件 | 規則 | 備註 |
|---|---|---|
| 目標格 | 必須是「土地」且尚未被任何房屋/田地/城堡/巢穴/橋/柵欄佔據 | |
| 周圍八格 | 我方既有房屋的 Moore neighborhood（8 格）內不得建房 | 對手或中立國的房屋不限制我方建造 |
| 金錢 | 我方國庫 ≥ 100 金 | 建造瞬間扣除 |
| 部隊 | 必須有我方部隊「位於該目標格」執行建造 | |

### 1.2 部隊人力分配

建造時，執行建造的部隊會分人給新房子：

```
if 建造部隊人數 <= 200:
    新房子人數 = floor(建造部隊人數 / 2)
    建造部隊剩餘人數 = 建造部隊人數 - 新房子人數
else:
    新房子人數 = 100
    建造部隊剩餘人數 = 建造部隊人數 - 100
```

**邊界測試 case（必寫單元測試）：**
- 建造部隊 = 199 → 房子分 99，部隊剩 100
- 建造部隊 = 200 → 房子分 100，部隊剩 100
- 建造部隊 = 201 → 房子分 100，部隊剩 101

### 1.3 建造完成後

- 房子立刻屬於我方
- 在地圖上佔據 1 格
- 開始計入「周圍八格不能再建房」的限制
- 開始觸發田地擴張與人口增長邏輯（見 §2、§3）

## 2. 田地擴張（Field Expansion）

每隔一段時間，房子會嘗試把周圍八格中「不是我方田地」的格子轉為我方田地。

| 項目 | 規則 |
|---|---|
| 觸發 | 由房子主動觸發（不是部隊） |
| 成本 | 每轉化一格扣房子 10 人 |
| 目標 | 周圍八格中可轉為田地的格子（土地、敵方田地等，依實作決定範圍） |
| 邊界 | 房子人數不足以扣除時，跳過該次擴張 |

### 2.1 人口生成

當房子人數 ≥ 100 時：
- 房子產出一支 100 人的新部隊
- 房子人數扣 100
- 部隊出生位置：實作上通常選在房子相鄰格中我方領土的一格

> ⚠️ 這條和田地擴張共用同一個「房子人數池」，所以擴張頻繁時人口會被消耗，部隊產生變慢。

## 3. 人口增長（Population Growth）

### 3.1 增長公式

當稅率為 0 時：
```
每日增加人數 = 2 + 周圍八格中我方田地數
```

稅率每提高，增長速度變慢。**稅率 30% 時完全不再生成新部隊**（增長停滯）。

> 攻略中沒給出 0%~30% 之間的精確曲線，M2 實作前需要決定：線性遞減？分段？建議先用線性 `growth_rate = (30 - tax_rate) / 30` 當係數，之後再 tune。

### 3.2 稅收連通性（Castle Connectivity）

| 狀態 | 稅收 |
|---|---|
| 房子與城堡「相連」 | 正常按稅率收稅 |
| 房子與城堡「不相連」 | 稅率視為 0，無稅收 |

**相連的定義**：存在一條路徑，從房子出發到城堡，路徑上的每一格都是「我方田地或我方房屋」，且**不經過柵欄**。

> 注意：路徑不能穿過其他國家的房子或田地，也不能穿過任何柵欄（包含我方自己的柵欄）。

### 3.3 連通性計算建議

每次以下事件發生時，需要重算（或 incremental update）連通性：
- 我方/敵方建造或摧毀房屋
- 我方/敵方建造或摧毀田地
- 柵欄建造或被破壞
- 領土歸屬改變（國王被打倒、房屋易主）
- 城堡狀態改變

**實作建議**：用 BFS/DFS 從城堡出發，標記所有可達的我方房屋。地圖不大時可每次全算；大地圖再考慮 union-find 或增量更新。

## 4. 房屋摧毀（Destroy House）

> 攻略中房屋摧毀的相關規則散在「五、戰鬥」與「六、建設和破壞建築」中。

- 房子可被敵方部隊攻擊摧毀
- 摧毀過程中，攻擊部隊每回合固定損失 10 人
- 每回合造成的破壞量：S/M 級為 `floor(sqrt(部隊人數 / 100))`，L 級攻擊破壞力極高（攻略未給精確公式，可先用近似值）
- 房子有「耐久度」概念（攻略未明說房屋耐久度數值，M2 需自行定義；可以拿房子人數當耐久，與田地擴張、人口生成共用同一個池）

## 5. 資料模型建議

```typescript
// 房子實體
interface House {
  id: string;
  owner: PlayerId;        // 所屬國家
  position: { x: number; y: number };
  population: number;     // 房子人數，同時可作為耐久值
  connectedToCastle: boolean;  // 由連通性計算更新
  lastExpansionTick: number;
  lastGrowthTick: number;
}

// 建造命令
interface BuildHouseCommand {
  unitId: string;         // 執行建造的部隊
  position: { x: number; y: number };
}

// 驗證結果
type BuildValidation =
  | { ok: true }
  | { ok: false; reason:
      | 'TILE_OCCUPIED'
      | 'NEIGHBOR_HAS_OWN_HOUSE'
      | 'INSUFFICIENT_GOLD'
      | 'NO_UNIT_ON_TILE'
      | 'TILE_NOT_BUILDABLE'
    };
```

## 6. 單元測試清單（M1 必須涵蓋）

### 建造驗證
- [ ] 空地、無金錢 → `INSUFFICIENT_GOLD`
- [ ] 空地、有金錢、無部隊 → `NO_UNIT_ON_TILE`
- [ ] 空地、八格內有我方房屋 → `NEIGHBOR_HAS_OWN_HOUSE`
- [ ] 空地、八格內只有敵方房屋 → 允許建造
- [ ] 河流/岩漿格 → `TILE_NOT_BUILDABLE`
- [ ] 已有田地的我方格 → `TILE_OCCUPIED`

### 人力分配邊界值
- [ ] 部隊 = 1 → 房子分 0、部隊剩 1（注意這種情況是否該禁止）
- [ ] 部隊 = 199 → 99 / 100
- [ ] 部隊 = 200 → 100 / 100
- [ ] 部隊 = 201 → 100 / 101

### 田地擴張 & 人口
- [ ] 房子人數 < 10 → 跳過擴張
- [ ] 房子人數達 100 → 產生 100 人部隊，自身 -100
- [ ] 稅率 0、周圍 0 田地 → 每日 +2 人
- [ ] 稅率 0、周圍 8 田地 → 每日 +10 人
- [ ] 稅率 30% → 不再產生新部隊

### 連通性
- [ ] 房子直接相鄰城堡 → connected
- [ ] 房子靠田地路徑連到城堡 → connected
- [ ] 中間插入敵方田地 → disconnected
- [ ] 中間插入我方柵欄 → disconnected
- [ ] 城堡被摧毀 → 全部 disconnected
- [ ] 摧毀路徑中的關鍵田地 → 對應房子 disconnected
